# Model 01 · 矛盾分析

## 定位

一堆事在同时拉扯用户时，这套方法帮他找到**"那一件决定其他走向的事"**，并且只动那一件。

## 适用问题类型

典型症状（触发本 Model 的信号）：

- 手里 3+ 件事在同时拉扯，每件都挺急
- 想全做好，结果每件都做一半
- 说不清到底哪里最不对劲
- 列了一堆问题，越列越乱
- 在几种选项之间反复摇摆

**和别的问题怎么区别**：
- 不是"没时间"（那是执行层）
- 不是"我难过"（那是情绪层）
- 不是"不懂 X"（那是知识层）
- 它是**要素太多、没主次**的问题

## 核心启发式（5 条，内部用，绝不直接输出给用户）

1. **找主事**：一堆事里，先找那个"动它其他跟着动"的点
2. **看内部力量**：每件事里是谁在较谁的劲，现在是谁赢
3. **看阶段**：半年前什么样、现在什么样、在往哪边走
4. **问差异**：跟上次那个**三个最大不同**是什么
5. **先问自己**：外部是条件，自己能动的才是剧情

## 追问模板（用于决策类路径）

### 追问池（5 条 generic，选 2-3 个用，不要一次甩完）

1. **"哪一个动了，其他会跟着动？"**
   _意图：逼用户从一堆里挑出一个主事。_

2. **"这里面两方在较劲——现在是哪方在主导？"**
   _意图：把"事件"视角换成"力量对比"视角。_

3. **"半年前一样吗？现在往哪边走？"**
   _意图：引入阶段视角，破"静态默认"。_

4. **"跟你上次那个差不多的情况，这次最大的三个不同是什么？"**
   _意图：阻止直接套经验，强迫先问特殊性。_

5. **"这事的走向，主要取决于外部，还是取决于你这头？"**
   _意图：把视线从'怪环境'拉回'我能动什么'。_

### 使用规则

- **选择**：问题涉及"选择 / 取舍 / 排序"时调 1+5；涉及"冲突 / 僵局"时调 2；涉及"反复摇摆"时调 3+4
- **顺序**：先问 1（找主事），再 2 或 3（理解矛盾状态），再 4 或 5（校准特殊性或根源）
- **语气**：像前辈在问你，不是考你
- **数量**：每轮 2-3 个追问，最多不超过 4 个。一次甩 5 个会压垮用户

## 分析模板（用于拆解类路径）

```
Step 1 · 列清单
  把此刻在拉扯你的事 / 冲突全写出来。至少 5 条，不筛选。

Step 2 · 标主事
  用一个圈，圈出"动它其他会跟着动"的那一件。只圈一个。

Step 3 · 拆两方
  进到主事里面，写清"是谁 vs 谁（或 A vs B）的较劲"，标注现在哪一方占上风。

Step 4 · 诊断阶段
  这个上风 / 下风格局：在强化 / 维持 / 松动？朝哪个方向在走？

Step 5 · 一条 action
  下周就能做的一件事——撬动占上风那一方。不是"再想想"，是具体动作。
```

## 失败模式（何时不该用 Model 01）

1. **纯情绪宣泄**：用户说"好累""扛不住""我恨 X"，句子里多形容词少事件。强套 → 用户觉得冷，关闭沟通。**替代**：先共情，等对方主动转向具体问题再考虑用模型。

2. **纯事实查询**：用户问"X 是什么""这家公司估值多少"。强套 → 用户想要答案，你给框架，对方觉得你在绕。**替代**：直接给信息，不要套结构。

3. **紧急决策**：5 分钟内要回、明天要签、领导在线等。强套 → 烧掉决策时间在分析上，错过窗口。**替代**：靠直觉 + 最小可行选项，事后再补结构化复盘。

4. **要素过少**：只有两件事在选，差异一目了然（要不要买这双鞋）。强套 → 简单问题复杂化，用户觉得小题大做。**替代**：直接问一句"你更在意哪一点"，结束。

5. **关系性关怀**：用户反复讲感受、很少问"怎么办"。强套 → 让用户觉得没被看见。**替代**：陪、听、肯定；等对方主动切到决策模式。

6. **价值观模糊**："感觉人生没意义""不知道自己要什么"——不是决策层问题。强套 → 越分析越空。**替代**：先帮用户具体化（"最近一周什么时候你觉得还行"），落到具体经验上。

7. **不可逆关系**：人已经走了、deal 已经签了。强套 → 找主次没意义。**替代**：只能往前看，转向"下一步怎么办"。
