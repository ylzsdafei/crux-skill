# Model 02 · 调查研究（Groundtruth）

## 定位

决策前的信息校准器。防"拍脑袋 + 听风就是雨"两种翻车。**在 Model 01 之前或之后**都能用。

## 适用问题类型

典型症状（触发本 Model 的信号）：

- 用户说得出结论，但一问"你怎么知道的"答不上来
- 用户说得出"大家都说 X"，但数不出 3 个具体"大家"是谁
- 用户拿别人跑通的经验往自己身上套，没比过两边差异
- 用户手上材料全是书本 / 报告 / 别人转述，没自己下场看过
- 在"听说" vs "真看到"之间分不清

## 核心启发式（5 条，内部用，绝不直接输出给用户）

1. **信息分级**：每条依据标一手 / 二手 / 听说——**主导是哪级**？
2. **挖一个到底**：挖一个具体案例到底，胜过十个泛泛印象
3. **默认权威存疑**：书 / 报告 / 大家都说的，**先假定未必适合你**
4. **强列反例**：判断成形后，强找 3 个反例再复核
5. **亲历者优先**：问真在现场做过的人，不问转述者

## 追问模板（用于决策类路径，5 条 generic）

1. **"你这个判断是亲历的、听说的、还是推出来的？"**
   _意图：触发信息分级。逼用户把"我觉得"拆成可分级的依据。_

2. **"'大家都说 X'——你具体见过几个，叫得出名字吗？"**
   _意图：把"群体印象"拆成真实样本。数不出就是错觉。_

3. **"能给我一个具体例子吗，带时间、人、数字？"**
   _意图：把抽象判断落到一个挖到底的案例。_

4. **"你参考的那个方案 / 经验，跟你现在情况三个最大不同是什么？"**
   _意图：防照抄套壳。_

5. **"反过来想，如果你这个判断是错的，会是哪里翻车？"**
   _意图：拆确认偏差，逼用户主动产出反例。_

### 使用规则

- **触发场景**：用户给出判断前调 1；涉及"大家都" / "都在做"调 2；涉及抽象结论调 3；涉及"抄经验"调 4；涉及"判断已成形"调 5
- **顺序**：先 1（分级）→ 再 3 或 4（具体化 / 差异化）→ 最后 5（反例）
- **数量**：每轮 ≤ 3 个追问

## 分析模板（用于拆解类路径，5 步，主线：信息分级）

```
Step 1 · 信息分级
  把所有支持判断的依据列出来
  每条标：一手 / 二手 / 听说 / 推测
  主导是哪一级？
  输出：依据清单 + 占比

Step 2 · 反例检查
  强列 3 个"跟我判断相反"的情况
  这些反例能不能解释？
  输出：3 条反例 + 各自是否站得住

Step 3 · 缺口识别
  当前判断里哪些关键信息其实没拿到？
  哪些是"我以为我知道、其实没数"？
  输出：缺口清单

Step 4 · 最小数据需求
  决策前必须补的 3 个数据点
  每条说清：找谁问、问什么、多久能到手
  输出：3 个数据任务 + 负责人 + 时限

Step 5 · 行动建议
  三选一收口：
    - 继续（依据扎实、判断稳）
    - 暂停（先补数据再决定）
    - 推翻重做（反例站得住 + 缺口过大）
```

## 失败模式（何时不该用 Model 02）

1. **时间窗口 < 48 小时**：危机响应、竞品突袭、deadline 逼近。强套 → 调研没做完决策窗口就关了。**替代**：走决断（有限信息 + 风险对冲），事后再补调研。

2. **用户已深度调研**：用户自述"我看了几十篇、聊了十几个人"。强套 → 再调研是拖延，不是信息不足。**替代**：转向 Model 03 实事求是（Phase 3）——"敢不敢直面已有结论"。

3. **问题本质是情绪 / 勇气 / 资源**：用户表面问"该不该做"，深层是"我不敢 / 我没钱 / 我在拖"。强套 → 调研建议变成继续拖延的借口。**替代**：先识别真问题，调研不是答案。

4. **信息源同质化**：问了十个人都是同一个圈子、同一行业、同一立场。强套 → 再访谈是回音壁。**替代**：先换调研对象类型（找反对者、找局外人、找不同年龄层）再深访。

5. **完全陌生领域**：基础术语都没有，一上来就深调研。强套 → 问不出关键问题。**替代**：先建一次粗框架（读 3 篇综述 / 问 1 个入门级朋友）再正式调研。

6. **纯事实类问题**：用户问"X 是什么"——这是科普不是决策。**替代**：直接给信息。
