# 步骤 1 · 提炼心智模型（Q1-Q6 六问）

## 用途
用于把一篇思想方法论的原文，蒸馏成一份"心智模型"初稿。六问设计是让一篇长文产出可直接喂给 skill runtime 的内部资料，而不是读书笔记。

## 输入
- 原文（≥1 篇长文，允许配合 1-2 篇补源）
- 去人格化要求（见约束部分）

## Prompt 模板

> 下面给你一篇原文。不要复述、不要摘抄，用**你自己的话**回答以下 6 个问题，每问的答案要**具体、可操作**。

**Q1 · 适用问题类型**
这篇告诉你：当你面对**什么类型**的问题时，应该怎么做？
- 不要泛化成"复杂问题"这种空话
- 要说清：问题长什么样、症状是什么、跟别的问题区别在哪
- 字数 ≤100

**Q2 · 认知陷阱**
它警告你要避免哪几个**思维误区**？
- 列 3-5 个，每个一句话讲清
- 尽量贴近"普通人真的会犯"的错误，不要只是理论上的错误

**Q3 · 决策动作**
它给出哪几条**具体的决策动作**？
- 可执行：读完能照着做
- 不是抽象口号（"要实事求是"不算）
- 列 3-6 条

**Q4 · 反面教材**
它反对什么做法？（与 Q3 对照）
- 举例或模式，不要只列名词

**Q5 · 现代场景启发式（5 条）**
把原文放到 2020s 的**商业 / 技术 / 人生**决策里，哪 5 条最直接可用？
- **剔除时代局限**：革命 / 阶级斗争 / 敌我二元不要出现（如果原文核心就是这个，挑它跨时代的壳）
- 每条 ≤25 字
- 风格："遇到 X，考虑 Y"

**Q6 · 失败模式**
这套方法什么时候**不该用**？
- ≥3 条
- 要具体：哪类问题会被套坏、套了会导致什么结果

## 输出要求
- 严格**不引用原文**（"原文说"/"作者认为"都不行）
- 严格**不用黑名单词**（见 skill 设计 §6）
- 意思必须你自己的话讲

## 检查清单
- [ ] Q1 是否具体化（不是"复杂问题"这种空话）
- [ ] Q2-Q3 对得上（陷阱和动作是否呼应）
- [ ] Q5 剔除了所有时代词
- [ ] Q6 不是场面话，每条都指向真实失败场景
