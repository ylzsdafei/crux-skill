# Model 02 · 调查研究（Groundtruth）· v1

## 定位（一句话）

决策前的信息校准器。防"拍脑袋 + 听风就是雨"两种翻车。

---

## 步骤 1 产出：心智模型

### Q1 适用问题类型

当你要做一个**有分量的判断**、但信息多来自"听说 / 大家觉得 / 我感觉"而不是亲历数据时。

典型症状：
- 你说得出结论，但一问"你怎么知道的"答不上来
- 你说得出"大家都说 X"，但数不出 3 个具体"大家"是谁
- 你拿别人跑通的经验往自己身上套，但没比过两边差异
- 你手上材料全是书本 / 报告 / 别人转述，没自己下场看过

### Q2 认知陷阱

1. **凭印象当事实**：几个片段 + 一点听说 + 自己加工 = "我心里有数"，其实没数
2. **把"都这么说"当证据**：信息源是同一批人或同一个圈子，样本自相关
3. **崇拜成功方案**：别人跑出来的打法默认就能复制，忘了比条件差异
4. **只看扎眼的极端案例**：新闻 / 朋友圈 / 爆款故事 → 以为是普遍情况
5. **决策在前、调研在后**：心里已经选好，调研只用来找支持证据
6. **觉得自己啥都懂，懒得下场**：位子越高越不想亲自问细节

### Q3 决策动作

1. **信息分级**：每条依据标明一手 / 二手 / 道听途说。主导依据是哪级？
2. **解剖一只麻雀**：找 1-3 个典型案例，挖到"时间、人、数字"级别
3. **问亲历者**：不问评论者、不问转述人，问真在现场做过的人
4. **列具体反例**：强迫自己想 3 个"跟我判断相反"的情况
5. **先做提纲再去问**：带着具体问题去谈，不是漫谈凑素材
6. **亲身下场至少一次**：不能全靠书面汇报 / PPT / 二手数据

### Q4 反面教材

1. 开会拍脑袋，没人带数据
2. "我觉得市场需要" / "用户都说需要" —— 没有具体几个用户、几笔数据
3. 抄别人的方案，没看过别人具体怎么跑的
4. 照搬书 / 报告 / 课程的结论，不比自家情况
5. 懒得做调查，只用"想当然"补信息缺口
6. 汇报时堆案例堆统计数字，但没有任何一个例子讲到底

### Q5 现代场景启发式（5 条）

1. 遇到"大家都说"，先数出具体几个人
2. 遇到"亲历 vs 听说"混着，拆开分别掂量
3. 遇到抄方案，先列三个最大差异
4. 遇到结论含糊，挖一个具体案例到底
5. 遇到判断已成形，强找三个反例

### Q6 失败模式

1. **紧急决策不用**：时间窗口 < 48 小时，调研会拖死行动
2. **完全陌生领域不要一上来就调研**：基础术语都没有，先建一次粗框架再调研
3. **问题本质不是信息问题**：用户怕 / 懒 / 拖 / 没资源，调研只是借口
4. **信息源已经饱和**：用户已经看了几十篇、聊了十几个人，再调研是拖延
5. **纯事实类问题**：用户问"X 是什么"，这是科普不是决策

## 步骤 2 产出：启发式提炼

### 候选清单（扫原文 + 扩写，25 条）

1. 没调查别开口
2. 调查本身就是在解决问题
3. 所有结论放到调查末尾，不放开头
4. 不要一到现场就下断语
5. 脑子空着出门，回来就有料
6. 开会问一群人，比单独听一个人靠谱
7. 开会问的人要混合年龄和身份
8. 会前要定好要问什么，不是漫谈
9. 亲自下场，别只看书面汇报
10. 第一回调研要深挖一个点
11. 自己做记录，不要代笔
12. 对书 / 报告 / 上级指示先假定"未必适合我的情况"
13. 别把"上面说"当免检
14. 搞清关键对象的构成（各类多少、各占多少）
15. 搞清关键对象之间的关系，不只是单体
16. 不调查的计划注定会错
17. 决策前必须先掌握情况，再制定策略
18. 别只看漂亮的极端案例
19. 别把听了新鲜故事当调研
20. 材料要成系统，不是零碎
21. 历史 + 现状同时看，不割断
22. 要看城市也要看乡村（引申：不偏食信息源）
23. 闭着眼睛想办法，一定出错办法
24. 不同立场的人对同一件事看法不同，要都听听
25. 光有热情没客观情况，一定翻车

### 合并去重（13 条）

1. 没调查别开口（合 1+23）
2. 结论要放在调研之后（合 3+17）
3. 不要一到就下断语（4）
4. 出门前带问题，回来有料（5）
5. 群访 > 独访，混身份（合 6+7+24）
6. 会前定纲目，不要漫谈（8）
7. 自己下场 + 自己记录（合 9+11）
8. 第一次调研要深挖一个点（10）
9. 对书 / 报告 / 权威默认存疑（合 12+13）
10. 搞清对象构成和对象之间的关系（合 14+15）
11. 不偏食信息源（合 22+24）
12. 材料要系统，不要零碎（合 20+19）
13. 历史 + 现状同看（21）

### 筛选：5 条核心启发式（入 Model 02）

维度评分都 ≥4 分、直接对接信息校准定位的五条：

1. **信息分级**：每条依据标一手/二手/道听途说，看主导级别
2. **解剖一只麻雀**：挖一个具体案例到底，胜过十个泛泛印象
3. **反对本本主义**：书本 / 报告 / 大家都说的，先默认未必适合你
4. **三个反例**：判断成形后，强找三个反例再复核
5. **亲历者优先**：找真在现场做过的人，不找转述者

### 筛选：3-5 条通用启发式（入 universal pool）

适合所有 Model 共用：

1. 没调查就没发言权
2. 结论放在过程末尾，不放开头
3. 先掌握情况再制定策略
4. 脑子空着出门，回来就有料

## 步骤 3 产出

### A. 追问模板（5 条，generic）

> 以下每条后面标"意图"，这是**内部注释**，不对用户输出。

1. **"这个判断你是怎么得出来的，亲身经历的、听人说的、还是自己推的？"**
   - 意图：触发信息分级启发式。逼用户显式区分一手/二手/推测
   - 用户典型反应：会停一下，承认"其实没那么扎实"

2. **"'大家都说需要'——你具体见过几个，是谁？"**
   - 意图：触发"数出具体人"启发式。把"群体印象"拆成真实样本
   - 用户典型反应：能数出 2-3 个就算有数据，数不出就是错觉

3. **"能给我一个具体例子吗，带时间、人物、数字。"**
   - 意图：触发解剖麻雀启发式。把抽象判断落到具体场景
   - 用户典型反应：讲不出细节 → 判断其实没落地

4. **"你参考的那个案例 / 方案 / 经验，跟你现在情况三个最大不同是什么？"**
   - 意图：触发反对本本主义启发式。防套壳
   - 用户典型反应：说不出差异 → 照抄风险高

5. **"反过来想，如果你这个判断是错的，会是因为什么？"**
   - 意图：触发三个反例启发式。拆支持偏差
   - 用户典型反应：沉默几秒 → 想到一个反例 → 判断松动

### B. 分析模板（5 步，主线是"信息分级"）

```
Step 1 · 信息分级
 列出所有支持你判断的依据
 每条标：一手 / 二手 / 道听途说 / 推测
 主导级别是哪一级？
 输出：依据清单 + 比例

Step 2 · 关键反例
 强迫自己列 3 个"跟我判断相反"的情况
 这些反例你能不能解释？
 输出：3 条反例 + 是否站得住

Step 3 · 调研缺口
 当前判断里哪些关键信息其实没拿到？
 哪些是"我以为我知道、其实我不知道"？
 输出：缺口清单

Step 4 · 最小数据需求
 决策前必须补的 3 个最小数据点
 每条说清：找谁问、问什么、多久能到手
 输出：3 个数据任务，带负责人 + 时限

Step 5 · 行动建议
 看完前 4 步，输出三选一：
  - 继续（信息够、判断稳）
  - 暂停（先补数据再决定）
  - 推翻重做（反例站得住 + 缺口过大）
```

### C. 失败模式（4 条）

1. **时间窗口 < 48 小时**：不走 Model 02，走决断；调研会拖死
2. **用户已经做过深度调研**：再调研是拖延，改走实事求是（敢不敢直面结论）
3. **问题本质是情绪 / 勇气 / 资源**：调研是借口，别让用户误以为是信息问题
4. **信息源同质化严重**：再问十个人也是同一个声音，得换调研对象类型
