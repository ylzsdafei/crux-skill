## 步骤 1 产出：心智模型

### 定位（一句话）
一堆事在同时拉扯一个人时，这套方法帮他找到**"那一件决定其他走向的事"**，并且只动那一件。

---

### Q1 · 适用问题类型

典型症状：
- 手里 3+ 件事在同时拉扯，每件都挺急
- 想全做好，结果每件都做一半
- 说不清到底哪里最不对劲
- 列了一堆问题，越列越乱
- 在几种选项之间反复摇摆

**和别的问题怎么区别**：
- 不是"没时间"（那是执行层）
- 不是"我难过"（那是情绪层）
- 不是"不懂 X"（那是知识层）
- 它是**要素太多、没主次**的问题

---

### Q2 · 认知陷阱

1. **平均用力**：把所有事看得一样重，分不出轻重。
2. **只看一面**：只看自己这头、只看当下、只看表面。
3. **把经验当公式**："上次这样解决的，这次也这样"，不问条件差异。
4. **推责任给环境**：出问题先找外部原因，忽略自己能改的部分。
5. **当成静态**：默认"事情就这样了"，忘了半年会有变化。
6. **抽象化逃避**：把具体事升级成"当代人的普遍困境"来讨论。

---

### Q3 · 决策动作（5 步）

1. **列清单**：把拉扯你的事全写下来（至少 5 条）。
2. **标主事**：哪一件**动了它其他会连带动**？只标一件。
3. **拆两方**：主事里本质是谁和谁的较劲？现在是哪一方占上风？
4. **问阶段**：这个格局半年前一样吗？现在在往哪边走？
5. **出一个动作**：下周就能做的一件事——**撬动占上风那一方**。

---

### Q4 · 反面教材

- **每件沾一手**：三个项目都推 30%，没一个成。
- **先开会讨论**：罗列了十个问题，开三次会，没处理掉一个。
- **照搬上次**：上次管用的招直接套，不问条件变了没。
- **哭诉外部**：市场不行 / 老板不行 / 队友不行，没想自己能动什么。
- **抽象化逃避**：把"和室友怎么谈"升级成"当代年轻人的社交困境"。
- **空转理论**：开口就是"要系统看、要全面看"，具体是什么事没说清。

---

### Q5 · 现代场景启发式（5 条）

1. **找主事**：一堆事里，先挑那个"动它其他跟着动"的点。
2. **看力量**：每件事背后是两股力在较劲，先问现在是哪股赢。
3. **看阶段**：半年一回望——卡住的格局在强化还是松动？
4. **问差异**：别套上次，先问这次跟上次**三个最大不同**。
5. **先问自己**：外部是条件，自己这头的动作才是剧情。

---

### Q6 · 失败模式

1. **纯情绪场景**：用户在宣泄，不是要决策。先共情。
2. **纯事实场景**：用户要答案，不是要框架。直接答。
3. **紧急决策**：5 分钟内要回复，靠直觉，别铺分析。
4. **要素少于 2 件**：简单问题会被这套工具变复杂。
5. **价值观模糊场景**："感觉人生没意义"这类，不是决策问题。先帮用户具体化。
6. **不可逆关系**：人已经走了、deal 已经签了，再找主次没意义，只能往前看。

### 筛选：5 条核心启发式（入 Model 01）

这 5 条之所以入选：
- **能直接变成一句话追问**
- **彼此不重复**（每条切一个独立维度：选点 / 看内部 / 看时间 / 避免套路 / 把视线转回自己）
- **贴 Model 01 的定位**：一堆事里找那一件

1. **找主事**：一堆事里，先找那个"动它其他跟着动"的。
2. **看内部力量**：每件事里是谁在较谁的劲，现在是谁赢。
3. **看阶段**：半年前什么样、现在什么样、在往哪边走。
4. **问差异**：跟上次那个**三个最大不同**是什么。
5. **先问自己**：外部是条件，自己能动的才是主角。

### 筛选：4 条通用启发式（入 universal-heuristics.md）

这 4 条跨 Model 都要用，不是 Model 01 专属：

1. **具体优先**：别把一件具体事升级成普遍问题再讨论。
2. **事情在变**：任何判断都有保质期，别把现在当永恒。
3. **两面都要看**：看 A 也看 B，看自己也看对方。
4. **不平衡是常态**：两方势均力敌只是瞬间，总有一头强一头弱。

## 步骤 3 产出

### A. 追问模板

#### 对 5 个测试题的具体追问

**① 手上 3 个项目顾不过来，该放弃哪个？**
- "3 个里头，**哪一个做起来了，另两个会自动不那么紧**？"
- "每个你最卡的那步——**是资源不够，还是你其实不想做**？"
- "半年前这 3 个同样重要吗？**什么时候开始不平衡的**？"
- "你上次同时扛多件事、最后抓了主做成的那回，**这回最大的不一样在哪**？"

**② 和室友闹矛盾要不要摊牌？**
- "你们摩擦——**是利益（水电/卫生）、习惯不同，还是积了很久的气**？三种的摊牌方式完全不一样。"
- "现在的关系里——**是你更憋，还是 TA 更憋**？TA 觉得自己输在哪？"
- "这摩擦是最近的事，还是一直有，最近你才扛不住？"
- "**你自己这头有没有能先动的**？还是一定要等 TA 先动？"

**③ 现有系统是修 bug 还是重构？**
- "最让你痛的那个 bug——**它修好了，重构的压力还剩多少**？"
- "你想修还是想重构——**是代码真烂，还是你对这套设计已经不信任了**？"
- "半年前同样的 bug 你是咋处理的？**现在为啥扛不住了**？"
- "重构这决定，**是技术理由，还是你想换种工作方式**？"

**④ 扩张速度太慢要不要改 to C？**
- "慢——**是 to B 本身就这速度，还是某一个环节卡了**？卡哪里？"
- "如果 to B 这条路**这个月就彻底卡死**，你会怎么办？**那个答案才是你真正的 plan B**。"
- "对比你刚开始做的时候，**市场信号是在好转还是变差**？"
- "to C 不是换口号，是换整套打法——**你团队里有谁是真会做 C 的**？"

**⑤ 今年最重要的是考证还是做副业？**
- "考证和副业——**哪一个做成了，另一个会变得不那么重要**？"
- "你想考 / 想做——**是被外部推（父母、同事都在），还是你真想要那个结果**？"
- "去年这个时段，你是怎么在这两件事之间分的？**今年为啥特别纠结**？"
- "这两件事背后你真正要的是啥——**安全感、钱、还是换轨道**？真要的东西决定优先级。"

#### 归纳出的 5 条 generic 追问模板

> 每条都能套到同类决策问题上。标注的"意图"是内部使用，用户看不到。

1. **"哪一个动了，其他会跟着动？"**
   _意图：逼用户从一堆里挑出一个主事。_

2. **"这里面两方在较劲——现在是哪方在主导？"**
   _意图：把"事件"视角换成"力量对比"视角。_

3. **"半年前一样吗？现在往哪边走？"**
   _意图：引入阶段视角，破"静态默认"。_

4. **"跟你上次那个差不多的情况，这次最大的三个不同是什么？"**
   _意图：阻止直接套经验，强迫先问特殊性。_

5. **"这事的走向，主要取决于外部，还是取决于你这头？"**
   _意图：把视线从"怪环境"拉回"我能动什么"。_

### B. 分析模板（拆解路径，通用 5 步）

```
1. 列清单
   把此刻在拉扯你的事 / 冲突全写出来。至少 5 条，不筛选。

2. 标主事
   用一个圈，圈出"动它其他会跟着动"的那一件。只圈一个。

3. 拆两方
   进到主事里面，写清"是谁 vs 谁（或 A vs B）的较劲"，标注现在哪一方占上风。

4. 诊断阶段
   这个上风 / 下风格局：在强化 / 维持 / 松动？朝哪个方向在走？

5. 一条 action
   下周就能做的一件事——撬动占上风那一方。不是"再想想"，是具体动作。
```

### C. 失败模式（≥3 条；每条含"场景 + 后果 + 替代"）

1. **情绪宣泄场景**
   - 场景：用户说"我今天好累"、"我真的扛不住了"、"我恨这份工作"，句子里多形容词少事件。
   - 强套后果：用户觉得冷，关闭沟通。
   - 替代：先接住情绪（"听起来这阵子很压"），等对方主动转向具体问题再考虑用模型。

2. **事实查询场景**
   - 场景：用户问"X 是什么"、"这家公司估值多少"、"某个技术靠谱吗"。
   - 强套后果：用户想要答案，你给框架，对方觉得你在绕。
   - 替代：直接给信息 / 答案，不要套结构。

3. **紧急决策场景**
   - 场景：5 分钟内要回、明天就要签、领导在线等。
   - 强套后果：把决策时间烧在分析上，错过窗口。
   - 替代：靠直觉 + 最小可行选项，事后再补结构化复盘。

4. **要素过少场景**
   - 场景：只有两件事在选，且差异一目了然（要不要买这双鞋）。
   - 强套后果：简单问题被复杂化，用户觉得你小题大做。
   - 替代：直接问一句"你更在意哪一点"，结束。

5. **关系性关怀场景**
   - 场景：用户反复讲自己的感受、很少问"怎么办"，要的是被理解 / 被肯定。
   - 强套后果：结构化分析让用户觉得没被看见。
   - 替代：陪、听、肯定；等用户主动切到决策模式再进工具。

6. **价值观模糊场景**
   - 场景："感觉人生没意义"、"不知道自己要什么"——这不是决策层的问题。
   - 强套后果：越分析越空，找不到那个"主事"，因为根本还没定义什么算"好"。
   - 替代：先帮用户具体化（"最近一周什么时候你觉得还行"），落到具体经验上再看能不能做决策。
