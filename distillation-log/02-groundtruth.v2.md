# Model 02 · 调查研究（Groundtruth）· v2

## 定位（一句话）

决策前的信息校准器。防"拍脑袋 + 听风就是雨"两种翻车。

---

## 步骤 1 产出：心智模型

### Q1 适用问题类型

当你要做一个**有分量的判断**、但信息多来自"听说 / 大家觉得 / 我感觉"而不是亲历数据时。

典型症状：
- 你说得出结论，但一问"你怎么知道的"答不上来
- 你说得出"大家都说 X"，但数不出 3 个具体"大家"是谁
- 你拿别人跑通的经验往自己身上套，但没比过两边差异
- 你手上材料全是书本 / 报告 / 别人转述，没自己下场看过

### Q2 认知陷阱

1. **凭印象当事实**：几个片段 + 一点听说 + 自己加工 = "我心里有数"，其实没数
2. **"都这么说"当证据**：信息源是同一个圈子，样本自相关
3. **崇拜成功方案**：别人跑出来的打法默认就能复制，忘了比条件差异
4. **极端案例错觉**：新闻 / 朋友圈 / 爆款故事被当普遍情况
5. **倒置的调研**：心里已经选好，调研只用来找支持证据
6. **位子越高越不问细节**：嫌屈尊，懒下场

### Q3 决策动作

1. **信息分级**：每条依据标一手 / 二手 / 听说。主导是哪级？
2. **挖一个到底**：找 1-3 个典型案例，挖到时间、人、数字
3. **问亲历者**：问真在场做过的，不问评论的、不问转述的
4. **强列反例**：强迫自己列 3 个"跟我判断相反"的情况
5. **先定提纲再问**：带具体问题去谈，不是漫谈凑料
6. **至少下场一次**：不能全靠 PPT / 汇报 / 二手数据

### Q4 反面教材

1. 开会拍脑袋，没人带数据
2. "我觉得" / "用户都说" —— 没有具体几个、几笔
3. 抄别人方案，没看过别人具体怎么跑
4. 照搬书 / 报告 / 课程结论，不比自家情况
5. 懒得问，用"想当然"补信息缺口
6. 汇报堆案例堆数字，却没有一个讲到底

### Q5 现代场景启发式（5 条）

1. 遇到"大家都说"，先数出具体几个人
2. 遇到亲历和听说混着，拆开单独掂量
3. 遇到抄方案，先列三个最大差异
4. 遇到结论含糊，挖一个案例到底
5. 遇到判断已成形，强找三个反例

### Q6 失败模式

1. **紧急决策不用**：时间窗口 < 48 小时，调研会拖死行动
2. **完全陌生领域不要一上来就调研**：基础术语都没有，先建一次粗框架再调研
3. **问题本质不是信息问题**：用户怕 / 懒 / 拖 / 没资源，调研是借口
4. **信息源已经饱和**：再调研是拖延
5. **纯事实类问题**：用户问"X 是什么"，这是科普不是决策

## 步骤 2 产出：启发式提炼

### 候选清单（25 条，扫原文 + 扩写）

1. 没摸清情况别下结论
2. 调查本身就是在解决问题
3. 结论放在调研末尾，不放开头
4. 不要一到就下断语
5. 脑子空着出门，回来就有料
6. 群访 > 独访
7. 群访对象混合年龄和身份
8. 会前定好要问什么，不漫谈
9. 亲自下场，别只看书面汇报
10. 第一回调研要深挖一个点
11. 自己做记录，不假手于人
12. 对书 / 报告 / 权威先假定"未必适合我"
13. 别把"上面说"当免检
14. 搞清对象内部构成（各类多少、各占多少）
15. 搞清对象之间的关系
16. 不调查就做计划，一定出错
17. 决策前先掌握情况，再定策略
18. 别只看漂亮的极端案例
19. 别把听了新鲜故事当调研
20. 材料要成系统，不是零碎
21. 历史 + 现状同时看
22. 信息源不要偏食
23. 闭着眼睛想办法，只能出错办法
24. 不同立场的人对同一件事看法不同，都要听
25. 光有热情没客观情况，一定翻车

### 合并去重（13 条）

1. 没摸清情况别下结论
2. 结论放在调研末尾，不放开头
3. 不要一到就下断语
4. 出门前带问题，回来有料
5. 群访 > 独访，并且混身份
6. 会前定纲目，不漫谈
7. 自己下场 + 自己记录
8. 第一次调研要深挖一个点
9. 对书 / 报告 / 权威默认存疑
10. 搞清对象构成和对象之间的关系
11. 信息源不要偏食
12. 材料要系统，别零碎
13. 历史 + 现状同看

### 筛选：5 条核心启发式（入 Model 02）

三维评分都 ≥4 分、直接对接"信息校准器"定位：

1. **信息分级**：每条依据标一手 / 二手 / 听说，看主导级别
2. **挖一个到底**：挖一个具体案例到底，胜过十个泛泛印象
3. **默认权威存疑**：书 / 报告 / 大家都说的，先假定未必适合你
4. **强列反例**：判断成形后，强找三个反例再复核
5. **亲历者优先**：问真在现场做过的人，不问转述者

### 筛选：4 条通用启发式（入 universal pool）

适合所有 Model 共用的 Model 02 贡献：

1. 没摸清情况先别下结论
2. 结论放在过程末尾，不放开头
3. 先掌握情况再定策略
4. 脑子空着出门，回来就有料

## 步骤 3 产出

### A. 追问模板（5 条，generic）

> "意图"是**内部注释**，不对用户输出。

1. **"你这个判断是亲历的、听说的、还是推出来的？"**
   - 意图：触发信息分级。逼用户把"我觉得"拆成可分级的依据
   - 用户典型反应：会停顿，承认"其实没那么扎实"

2. **"'大家都说需要'——你具体见过几个，叫得出名字吗？"**
   - 意图：把"群体印象"拆成真实样本。数不出就是错觉
   - 用户典型反应：能数出 2-3 个算有根据，数不出就是脑补

3. **"能给我一个具体例子吗，带时间、人、数字？"**
   - 意图：把抽象判断落到一个挖到底的案例
   - 用户典型反应：讲不出细节 → 判断其实没落地

4. **"你参考的那个方案 / 经验，跟你现在情况三个最大不同是什么？"**
   - 意图：防照抄套壳
   - 用户典型反应：说不出差异 → 抄过去大概率翻车

5. **"反过来想，如果你这个判断是错的，会是哪里翻车？"**
   - 意图：拆确认偏差，逼用户主动产出反例
   - 用户典型反应：沉默几秒 → 想到一个 → 判断松动

### B. 分析模板（5 步，主线：信息分级）

```
Step 1 · 信息分级
 把所有支持判断的依据列出来
 每条标：一手 / 二手 / 听说 / 推测
 主导是哪一级？
 输出：依据清单 + 占比

Step 2 · 反例检查
 强列 3 个"跟我判断相反"的情况
 这些反例能不能解释？
 输出：3 条反例 + 各自是否站得住

Step 3 · 缺口识别
 当前判断里哪些关键信息其实没拿到？
 哪些是"我以为我知道、其实没数"？
 输出：缺口清单

Step 4 · 最小数据需求
 决策前必须补的 3 个数据点
 每条说清：找谁问、问什么、多久能到手
 输出：3 个数据任务 + 负责人 + 时限

Step 5 · 行动建议
 三选一收口：
  - 继续（依据扎实、判断稳）
  - 暂停（先补数据再决定）
  - 推翻重做（反例站得住 + 缺口过大）
```

### C. 失败模式（4 条）

1. **时间窗口 < 48 小时**
   - 场景：危机响应、竞品突袭、deadline 逼近
   - 后果：调研没做完决策窗口就关了
   - 替代：走决断（以有限信息 + 风险对冲）

2. **用户已经做过深度调研**
   - 场景：用户自述"我看了几十篇、聊了十几个人了"
   - 后果：再调研是拖延，不是信息不足
   - 替代：走实事求是（敢不敢直面已有结论）

3. **问题本质是情绪 / 勇气 / 资源**
   - 场景：用户表面问"该不该做"，深层是"我不敢 / 我没钱 / 我在拖"
   - 后果：Model 02 给出的调研建议会被用户用来继续拖延
   - 替代：先识别真问题，调研不是答案

4. **信息源同质化**
   - 场景：问了十个人都是同一个圈子、同一个行业、同一种立场
   - 后果：再访谈也是回音壁
   - 替代：先换调研对象类型（找反对者、找局外人、找不同年龄层）

## v1 → v2 改动记录

1. 启发式 3 "反对本本主义" → "默认权威存疑"
   - 原因：黑名单词规避 + 更直白
2. 追问 1 "亲身经历的、听人说的、还是自己推的" → "亲历的、听说的、还是推出来的"
   - 原因：精简
3. 启发式名称全部精简为 3-5 字短句（挖一个到底 / 强列反例 / 亲历者优先）
4. 失败模式加了"场景 / 后果 / 替代"三要素，每条更可判断
5. 分析模板 Step 命名从"调研缺口 / 最小数据需求"合并为"缺口识别 / 最小数据需求"（更好读）
